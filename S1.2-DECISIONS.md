## D3 — staleness check resolution
Plan 01 §"Brain-lint design" §checkStaleness mandates: "resolves the project root via
`brain/projects/<name>/profile.md` → cited project repo, then runs `git log` against
*that* tree". Implemented:
- Theme files under `brain/projects/<name>/themes/<file>.md` use that project's repo.
- Project repo path is resolved via `<forgeRoot>/projects/<name>/` (forge convention).
- If the project repo does not exist (e.g. unmanaged project), staleness is **skipped**
  for that theme (warn-only) — we cannot prove staleness without a tree to query.
- For *forge*-level themes (`brain/forge/themes/`) the staleness check runs against the
  forge worktree itself.

## D4 — `## Sources` link parsing
Treated as plain Markdown:
- Links of shape `[text](path)` whose path is relative — resolved relative to the theme file.
- Links of shape `[[wikilink]]` — resolved as a theme slug under `brain/forge/themes/`
  or `brain/projects/<scope>/themes/` (try both).
- Links to external URLs (`http(s)://`) and anchor-only links (`#section`) skipped.

## D5 — `checkOrphans` semantics
A theme is "reachable" iff its slug appears as a link target on **any** of:
- `brain/INDEX.md`
- `brain/forge/<category>.md`
- `brain/projects/<project>/<category>.md`
- `brain/projects/<project>/profile.md`
README.md theme files are exempt (they are templates, not real themes).

## D6 — `checkLengthSoftCap` line counting
Line-count == number of `\n` in the file (raw, including frontmatter). 60 lines warn,
100 lines error — matching `brain/LINT.md`.

where one's filename contains `-antipattern` (or category=`antipattern`) and the other's
contains `-pattern` (or category=`pattern`), surface as a flag. Never an error.
Per plan 01 explicit downgrade (council 01 flag).

## D17 — root cause discovered while writing the boundary test
The 128 contamination dirs were NOT due to crashed test runs — they were
caused by a **bug** in `restoreLiveBrain` itself: `rmSync(dir, {recursive: false})`
raises EISDIR on directories, silently failing every cleanup. Fixed to
`recursive: true`. The boundary test now both **proves the fix** and
exercises the **pre-clean fallback** for any directories left by genuinely
interrupted runs going forward.
