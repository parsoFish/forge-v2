## Conservative compression policy

Per the constraints above, this proposal is INTENTIONALLY conservative:

- Status entries in CLAUDE.md (the ✅ lines documenting closure passes)
  are NOT compressed. They're load-bearing audit trail; the operator
  reads them for cycle-trajectory context. Touching them would risk
  losing meaning.
- ADR refs, file paths, code blocks, mermaid diagrams: preserved
  byte-perfect.
- Headings, frontmatter, callout-block markers: preserved byte-perfect.
- The "you must" / "you must NOT" style preserved where the imperative
  is load-bearing (CLAUDE.md "Always do" / "Never do" lists).

The savings below are therefore lower-bound estimates. The operator
may choose to compress more aggressively on a per-paragraph basis after
the first wake-up review pass; that's expected iteration.
